Standard (noun)

  1. an accepted or approved example of something against which others are judged or measured
  2. (often plural) a principle of propriety, honesty, and integrity: "she has no standards"
  3. a level of excellence or quality: "a low standard of living"
  4. (...)

If Nothing Else. Please, Let there be Standards.

CSS Rules

Your standards should...

promote the same format and structure

/* Bad boy */
.selector-1[attr=value],
  .selector-2{
  background-color: white;
    color: black;
  margin:10px;
  padding:
    5px 10px;
    
  }/* Good girl */
.selector-1[attr="value"],
.selector-2 {
  background-color: white;
  color: black;
  margin: 10px;
  padding: 5px 10px;
}

reflect best practices

/* Bad boy */
.selector {
  background: #ffddcc;
  line-height: 1.2rem;
  margin: 0px !important;
}/* Good girl */
.selector {
  background-color: #fdc;
  line-height: 1.2;
  margin: 0;
}

define how to handle Vendor Prefixes

  • Use PostCSS' Autoprefixer or similar
  • Alternatively, if you have a preprocessor, use mixins
  • Last mesure, define browsers and create a table with required prefixes

guide the usage of Media Queries

.select-1 {
  /* Declarations */
}
@media (min-width: 800px) {
  .select-1 {
    /* Declarations */
  }
}/*
Don't shy away of
max-with rules
*/
@media (max-width: 480px) {
  .mobile-prevent-scroll {
    /* Declarations */
  }
}

prohibit the use of @import

  • Performance sucks
  • With preprocessors it's ok, they compile those rules
  • In CSS, use build tools to generate a single CSS file

Naming Conventions

Choose a structure that gives meaning to both markup and CSS

/* Bad boy */
.sf {
  /* Declarations */
}/* Good girl */
.search-form {
  /* Declarations */
}

Declare special classes

for specific uses with the help of "name spaces"

State Classes

/* Bad */
.is-error {
  border-color: red;
  color: red;
}/* Good */
.selector.is-error {
  border-color: red;
  color: red;
}

Provide separation of purpose

/* Great! But not here... */
.js-search-form__submit {
  border: 1px solid red;
}

Documentation

Go further when you NEED to

/* This is something to look out for. */
.selector {
  border: 1px solid red;
}

Go further when you HAVE to

.selector {
  /* Because of some DARN vendor code! */
  background-color: green !important;
}

Structure when and how to make shit obvious!

Use something like KSS or your very own brand of structure

/*
A button suitable for giving stars to someone.

:hover             - Subtle hover highlight.
.stars-given       - A star.
.stars-given:hover - Subtle hover star.
.disabled          - Dims stuff.

Styleguide 2.1.3.
*/
a.button.star{
  ...
}
a.button.star.stars-given{
  ...
}
a.button.star.disabled{
  ...
}

What do Standards give us?

  • The benefit of a head start*
  • Offer a base point*
  • They touch every bit of code
  • They make many sound like One
  • They offer easy access for every newcomer
  • They offer easy access to "old" code

So Standards are good...
